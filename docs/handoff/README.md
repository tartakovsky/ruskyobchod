# ruskyobchod handoff pack

Start here.

This folder exists because the live site was edited for weeks like a group project done in a moving car. Some of the changes fixed real problems. Some of them also created new ones. The next agent should not have to rediscover that from scratch.

## Reading order

1. `01-agent-git-workflow.md`
2. `02-issue-overview.md`
3. `07-current-live-state.md`
4. `03-current-state-and-experiments.md`
5. `04-architecture-review-and-rebuild-plan.md`
6. `05-quick-fix-plan.md`
7. `06-gls-regression-findings.md`
8. `08-next-phase-plan.md`

## What this pack is for

- establish one workflow for agent-driven changes
- summarize what the site history says went wrong
- capture the fixes and tests already done in the current incident
- separate proper architecture from emergency stabilization
- expose one authoritative current-state entry point before older incident notes

## What this pack is not

- not a complete site spec
- not a deployment system
- not a guarantee that the current live code matches this repo exactly

The repo is now the working control surface. The live Hostinger server is the thing you change only on purpose.
