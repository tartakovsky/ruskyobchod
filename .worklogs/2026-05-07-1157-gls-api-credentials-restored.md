# Restore GLS API credentials

## Context

Generating the GLS label for order `#11277` failed in the admin UI with:

`Password not set for GLS API`

The live option `woocommerce_gls_shipping_method_settings` had empty API credentials.

## Change

Restored the saved MyGLS Slovakia credentials into the live WooCommerce GLS settings:

- client number present
- username present
- password present
- country `SK`
- mode `production`
- account mode `single`

The credential values are intentionally not written into this worklog.

## Verification

Read back the live option after update and confirmed that client number, username, and password are set.

## Notes

No GLS label was generated by the agent during this step.
